# _setStartValueHMI.py

Write StartValue and Comment from an HMI Variables Excel file back into TIA Portal DB/IDB exports.

## Description

Reads the Excel file generated by `_extractHMI.py` and updates `Start value` and `Comment` in the corresponding source files. This is the reverse operation — edit values in Excel, then push them back into the TIA export files.

### Supported target formats

| Format         | Extension | Block type                                       |
| -------------- | --------- | ------------------------------------------------ |
| SimaticML XML  | `.xml`  | `SW.Blocks.GlobalDB`, `SW.Blocks.InstanceDB` |
| DB text source | `.db`   | Global Data Blocks                               |

### Update rules

- Only rows with a non-empty `Start value` or `Comment` column are processed.
- Virtual array-element rows (e.g. `DB.Array[0]`) are skipped — arrays are not expanded back.
- Only changed values are written; unchanged files are not modified.
- For XML files: updates `<StartValue>` and `<Comment><MultiLanguageText Lang="en-US">` on `<Member>` elements.
- For `.db` files: updates the `:= value` and `// comment` parts of member declaration lines.
- Blocks listed in Excel but not found in `Program blocks/` are reported as warnings.

## Usage

```powershell
# Auto-detect Excel and PLC folder
python Tools\_setStartValueHMI.py

# Explicit Excel file
python Tools\_setStartValueHMI.py <xlsx_file>

# Explicit Excel file and PLC folder
python Tools\_setStartValueHMI.py <xlsx_file> <plc_base_dir>
```

## Input

Default: `UserFiles\HMI_Variables_{plc_name}.xlsx` (same path as `_extractHMI.py` output).

The Excel file must have the same column layout as produced by `_extractHMI.py`:

| Col A | Col B     | Col C       | … | Col I   |
| ----- | --------- | ----------- | -- | ------- |
| Name  | Data type | Start value | … | Comment |

Columns **A** (Name), **C** (Start value), and **I** (Comment) are used.

## Dependencies

- `openpyxl` — Excel file reading
